문서 가이드

네이밍

기본규칙

  1. 이름은 영문 소문자, 숫자, 언더스코어(_) 또는 대시(-)로만 작성하고, 영문 대문자/숫자로 시작하지 않는다.
    - 클래스명 : 대시(-)로 구분
    - 파일명 : 언더스코어(_)로 구분
  2. 단어의 조합은 최대 3단계 이상을 넘지 않도록 한다.
  3. 숫자 앞에는 언더스코어를 붙이지 않으며, 숫자가 없으면 '1'이라는 숫자가 생략된 것으로 간주한다. 단, 숫자 정보이면 '1'을 표기한다.
  4. 2개 이상의 단어를 언더스코어없이 조합하고자 할 때 카멜표기법을 사용한다.
  5. 이미지명의 숫자 조합은 이미지 확장자와 관계없이 순차적으로 적용한다.

ID 및 CLASS 네이밍 규칙

  1. 요소의 ID속성는 문서에 한번만 사용한다.
  2. 레이아웃을 제외한 기타 요소에는 ID속성을 지정하지 않는다.
  3. 레이아웃 ID는 추가 단어의 조합을 하지 않고 Class명으로 구분한다.

id 및 class 약속어는 별첨




HTML 작성 규칙

기본규칙

  1. 모든 HTML 파일의 시작 부분에 문서의 DOCTYPE타입을 선언해야 하며, 기본 DTD는 html5을 사용한다.
  2. HTML의 모든 요소는 DOCTYPE 명세의 표준에 맞게 작성하며, W3C Validation검사를 통과해야 한다.
  3. 모든 요소와 속성는 소문자로 작성하며 속성값은 큰따옴표("")로 묶는다.
들여쓰기
  1. 마크업의 중첩이 깊어질 때마다 자식 요소는 1탭 들여쓰고, 1탭의 크기는 공백 4칸으로 설정한다.
    단, 다음과 같은 경우에는 들여쓰지 않는다.
    - <html>의 자식 요소: <head>, <body>
    - <body> 최상위 컨테이너 요소 : <div id=”wrap”>
    - <table>의 자식 요소인 <caption>,<colgroup>
    - <fieldset>의 자식 요소인 <legend>
  2. 자식요소 간 동일한 레벨의 요소는 들여쓰기를 하지 않는다.
주석
  1. HTML 주석의 시작과 종료 표기 방식
    - 시작주석 : <!-- 주석내용 -->
    - 종료주석 : <!-- // 주석내용 -->
  2. 최상위 컨테이너(id="wrap")을 제외한 레이아웃 요소와 독립된 콘텐츠 영역의 시작과 끝에는 주석을 표기하며, 시작/종료 주석의 내용은 동일하게 작성한다.


요소별 규칙

<title>
  1. 현재 페이지를 기준으로 메뉴 깊이의 역순으로 작성한다.
    브라우저 타이틀 제공 순서 : <title>현재페이지 타이틀 > 2Depth Title > 1Depth Title >사이트명</title>
  2. 혹은, 현재 페이지 | 사이트명
스킵 네비게이션
  1. 최상위 컨테이너 첫번째 자식요소로 선언하여, 키보드 접근시 '건너뛰기 링크'를 제공하도록 한다.
레이아웃 요소
  1. 화면 레이아웃을 나눌 때, 의미가 부여된 컨테이너 태그를 사용한다.
    - <header> : 배너, 머리말
    - <nav> : 내비게이션
    - <main> : 본문
    - <footer> : 바닥글, 콘텐츠 정보
    - <article> : 독립된 영역
    - <section> : 연관된 구문, 단락
  2. 구역을 나누는 구획 요소(Sectioning Elements)에는 제목 요소가 제공되어야 한다.
form
  1. 서로 관련있는 폼 컨트롤은 그루핑하여 <fieldset> 요소로 묶어준다. 그루핑 적용시, <legend>요소를 통해 필드셋 그룹에 대한 간단한 타이틀을 부여한다.
  2. *<fieldset> 요소는 중첩사용을 지양한다.
  3. <label>은 폼 컨트롤과 명시적으로 1:1 대응해야 한다.
table
  1. Table 첫 행에 <caption>요소를 사용하여 테이블의 간단한 정보를 추가해 준다.
  2. <th>요소에는 scope속성을 선언하여 지정된 셀의 머리글 정보를 행과 열의 데이터값과 매칭시켜 준다. 단, abbr속성의 사용은 선택사항이다.


CSS 작성 규칙

기본규칙

  1. CSS코드 작성시 들여쓰기는 하지 않는다.(단, 미디어쿼리 적용시 구간별 추가 스타일 속성은 1탭 들여쓰기로 구분한다.)
  2. 따옴표 사용 범위
    - 큰 따옴표("") : @charset선언
    - 작은 따옴표('') : 공백이 포함된 폰트명, 한글 폰트명, 데이터 타입, format, filter속성의 파라미터값
    - url데이터 타입, html속성에는 작은 따옴표를 사용하지 않는다.
  3. 스타일의 공통 속성 선택자 "*"의 사용을 지양한다.
  4. OS, Device, Browser 별 클래스네임
    - OS : mac / windows
    - Device : pc(desktop) / tablet / mobile
    - Browser : ie(edge) / chrome / firefox / safari / opera

속성

  1. font약식 속성은 사용하지 않는다.
  2. background 약식 순서 : color image position / size repeat attachment origin clip
  3. line-height의 요소 기본값에는 단위를 붙이지 않는다.
  4. 속성값이 0일 경우 단위를 표시하지 않는다.
  5. 문자로 표현한 위치값은 숫자로 변환한다.

단위 및 기본설정(root) 정의

  1. root 선언부의 폰트 기본단위는 rem으로 시작한다. (PC 작업기준)
    - font-size 기본값 = 0.625rem(10px)
    이하, 모든 스타일 속성값의 단위는 rem으로 치환하여 정의한다.

미디어쿼리

  1. 반응형 구간별 기준
    - PC(Desktop) : 저해상도 1024 ~ 1440 / 고해상도 1441 ~ 1920
    - Tablet : 768 ~ 1023
    - Mobile : 320 ~ 767
